### Change 1: Symbol Export Extraction (15 min)

**What:** Store exported symbols in `FileIndex` so the retrieval layer can match
symbol names to files.

**Why (value):**
- **$0 token cost** вҖ” extracted once during indexing, stored permanently
- **Enables every other retrieval improvement** вҖ” without symbol data, nothing else works
- **Makes the agent smarter** вҖ” "edit authenticate" finds `auth.ts` regardless of filename

**How:**
```typescript
// shared/types.ts: Add to FileIndex
export interface FileIndex {
  path: string
  skeleton: string
  imports: string[]
  exports: string[]        // вҶҗ NEW
  contentHash: string

// Each parser (TypeScriptParser, PythonParser, RustParser) already extracts
// symbol names in their parseFile() method вҖ” just store them.
// Example from TypeScriptParser.parseFile():
// Currently discards symbol names after building skeleton text.
// Fix: extract them into a Set during parsing, return in FileIndex.
```

**Files:** `shared/types.ts`, `parsers/typescript-parser.ts`, `parsers/python-parser.ts`, `parsers/rust-parser.ts`

### Change 2: Inverted Symbol Index (30 min)

**What:** Build a `symbol вҶ’ files[]` Map during graph construction so pi-slim can

**Why (value):**
- **~200t saved per lookup** вҖ” agent doesn't need to grep or guess file names
- **Zero runtime cost** вҖ” index built once at session start, cached with rest of `RepoIndex`
- **Foundation for intelligent retrieval** вҖ” every subsequent feature depends on this

**How:**
```typescript
// shared/types.ts: Add to RepoIndex

  deps: Map<string, Set<string>>

    symbolIndex.get(sym)!.push(f.path)
  }
}
```

### Change 3: Reverse Dependency Index (15 min)

**What:** Build a `file вҶ’ files_that_import_it[]` map so pi-slim can answer
"what depends on auth.ts?" and use it for retrieval ranking.

**Why (value):**
- **Better retrieval** вҖ” files imported by active files are more relevant, scored higher
- **Impact analysis** вҖ” "changing auth.ts will affect X, Y, Z"
- **Trivially computed** вҖ” just reverse the existing `deps` Map

**How:**
```typescript
// indexer/engine.ts: In buildGraph(), existing deps building loop:
for (const f of files) {

      // NEW: also populate reverseDeps

      reverseDeps.get(resolved)!.add(f.path)
    }
  }
}
```

### Change 4: Scoring + Retrieval Function (45 min)

**What:** Replace the regex-based file detection in `ContextInjector.detectInFocusFiles()`
with a scored retrieval function using all available signals.

**Why (value):**
- **20-40% better context relevance** вҖ” ranked files means the most important ones fit in budget
- **Eliminates false positives** вҖ” regex matches on file-like strings that don't actually exist
- **Extensible** вҖ” new signals (BM25, recency) can be added without changing the caller

**Score formula:**
```
score(file) =
  3 Г— symbolMatch(userMessage, file.exports) +      // "edit authenticate" вҶ’ auth.ts
  2 Г— filenameMatch(userMessage, file.path) +        // "look at auth.ts" вҶ’ auth.ts
  1 Г— dependencyProximity(file, activeFilesSet) +    // adjacent in dep graph
  1 Г— recencyBoost(file, recentlyEditedFiles)        // just modified вҶ’ higher
```

**Token savings:**
- Agent stops getting irrelevant files injected (like `logger.ts` when working on `auth.ts`)
- Files that match conversation intent ranked first вҶ’ fit in budget вҶ’ no budget waste

**How:**
```typescript
// NEW: context/retrieval.ts
export class RetrievalEngine {
  constructor(private index: RepoIndex) {}

  score(query: string, file: string, activeDeps: Set<string>): number {
    let score = 0
    const exports = this.index.symbolIndex.get(file) ?? []
    // Symbol match
    for (const sym of exports) {
      if (query.toLowerCase().includes(sym.toLowerCase())) score += 3
    }
    // Filename match
    const name = file.split('/').pop()?.replace(/\.[^.]+$/, '') ?? ''
    if (query.toLowerCase().includes(name.toLowerCase())) score += 2
    // Dependency proximity
    if (activeDeps.has(file)) score += 1
    return score
  }

  retrieveTopK(query: string, k: number = 20, activeDeps: Set<string> = new Set()): string[] {
    const candidates: Array<{ file: string; score: number }> = []
    for (const file of this.index.skeletons.keys()) {
      const s = this.score(query, file, activeDeps)
      if (s > 0) candidates.push({ file, score: s })
    }
    return candidates
      .sort((a, b) => b.score - a.score)
      .slice(0, k)
      .map(c => c.file)
  }
}
```

**Files:** `context/retrieval.ts` (NEW), `context/dep-context.ts` (integrate), `manager.ts` (wire up)

### Change 5: LSP вҶ’ Context Auto-Injection (20 min)

**What:** When the agent calls `lsp_go_to_definition` or `lsp_find_references`, extract
resolved file paths from the LSP result and auto-inject their skeletons in the next context.

**Why (value):**
- **~200t saved per LSP call** вҖ” eliminates the manual `read` call on the resolved file
- **Make LSP actually useful** вҖ” currently returns a string; now it feeds context
- **Natural bridge** вҖ” from "find where X is defined" to "here's the definition's skeleton"

**Current waste path:**
1. Agent calls `lsp_go_to_definition` вҶ’ gets `file:///src/auth.ts:10:5`
2. Agent calls `read` on `src/auth.ts` вҶ’ ~200 tokens burned
3. Agent sees skeleton вҶ’ can now work

**After fix:**
1. Agent calls `lsp_go_to_definition` вҶ’ pi-slim extracts path
2. pi-slim auto-injects `src/auth.ts` skeleton in next dep-context вҶ’ 0 extra tokens

**How:**
```typescript
// manager.ts handleContext: After scanning tool calls + output:
// NEW: Also scan for LSP results in recent tool calls
for (const msg of event.messages) {
  const toolName = (msg as Record<string, unknown>).toolName as string | undefined
  if (toolName === 'lsp_go_to_definition' || toolName === 'lsp_find_references') {
    const resultText = extractText((msg as Record<string, unknown>).content)
    // Extract file paths from LSP result format: "1. file:///src/auth.ts:10:5"
    const matches = resultText.matchAll(/file:\/\/([^:\n]+):\d+/g)
    for (const m of matches) {
      extraPaths.add(m[1])
    }
  }
}
```

**Files:** `manager.ts` (~15 lines)

### Change 6: Read вҶ’ Context Pre-Injection (15 min)

**Why (value):**

```typescript
      // Also add 1st-degree deps

        extraPaths.add(dep)
      }
    }
  }
}
```

### Change 8: Transitive Dependency Resolution (20 min)

**What:** Add a configurable `dependencyDepth` parameter that controls how many
levels of import chain are included in `<dep-context>`.

**Why (value):**
- **Better context for complex architectures** вҖ” LLM sees the full import chain
- **Controlled by budget** вҖ” depth naturally limited by `maxInjectionTokens`
- **Configurable** вҖ” small projects benefit from depth 2-3, large projects stay at 1

**How:**
```typescript
// context/dep-context.ts: In buildInjection(), after collecting 1st-degree deps:
if (this.config.dependencyDepth > 1) {
  let currentDeps = new Set(depPaths)
  for (let depth = 2; depth <= this.config.dependencyDepth; depth++) {
    const nextLevel = new Set<string>()
    for (const dep of currentDeps) {
      for (const subDep of index.deps.get(dep) ?? []) {
        if (!inFocus.has(subDep) && !depPaths.has(subDep)) {
          nextLevel.add(subDep)
        }
      }
    }
    for (const nd of nextLevel) depPaths.add(nd)
    if (nextLevel.size === 0) break // no more edges
    currentDeps = nextLevel
  }
}
```

**Files:** `context/dep-context.ts` (~10 lines), `context/schema.ts` (~1 line)

### Change 9: Repo Map Relevance Sorting (15 min)

**What:** Sort `<repo-map>` entries by recency (last git modification time) instead
of alphabetically, and cap entries to keep the most relevant.

**Why (value):**
- **~200t saved on large projects** вҖ” irrelevant files (migrations, old tests) pushed to bottom
- **Better first impression** вҖ” the agent sees the working set first
- **Git-driven** вҖ” files you just modified are likely the ones you're working on

**How:**
```typescript
// context/repo-map.ts: In generate(), sort files within each directory:
import { execSync } from 'node:child_process'

function getGitMtime(file: string): number {
  try {
    const output = execSync(`git log -1 --format=%at -- ${file}`, {
      cwd: projectRoot, stdio: ['pipe', 'pipe', 'ignore']
    }).toString().trim()
    return Number(output) || 0
  } catch { return 0 }
}

// Sort files within each dir by mtime descending:
fileLines.sort((a, b) => {
  const timeA = getGitMtime(join(projectRoot, dir, a.name))
  const timeB = getGitMtime(join(projectRoot, dir, b.name))
  return timeB - timeA
})
```

**Files:** `context/repo-map.ts` (~15 lines)

### Change 12: `/slim explain` Command (20 min)

**What:** Add a `/slim explain` command that shows WHY specific files were included
in the last dep-context injection.

**Why (value):**
- **Developer transparency** вҖ” understand the retrieval pipeline

```typescript
  included: boolean
  trimmed: boolean
}

// Store in session state for /slim explain to read

// Command handler:
pi.registerCommand('slim explain', {
  handler: async () => {
    const explanations = manager.state?.lastRetrievalExplanation
    for (const e of explanations ?? []) {
      const icon = e.included ? 'вң“' : e.trimmed ? 'вңӮ' : 'вң—'
      ctx.ui.notify(`${icon} ${e.file} (score: ${e.score.toFixed(1)}) signals: ${e.signals.join(', ')}`)
    }
  }
})
```
